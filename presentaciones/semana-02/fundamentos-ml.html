Qué es Machine Learning

La diferencia entre programar reglas y aprender patrones

Definición fundacional (Arthur Samuel, 1959)

"El campo de estudio que permite a las computadoras aprender sin ser programadas explícitamente."

Definición operativa (IBM, 2026)

"Subconjunto de la IA enfocado en algoritmos que pueden 'aprender' patrones de datos de entrenamiento y, posteriormente, hacer inferencias precisas sobre datos nuevos."

Definición formal (Tom Mitchell)

"Un programa aprende de la experiencia E con respecto a alguna tarea T y medida de rendimiento P, si su rendimiento en T, medido por P, mejora con E."

El concepto clave: GENERALIZACIÓN — la capacidad de traducir buen rendimiento en datos de entrenamiento hacia resultados útiles en escenarios del mundo real.

Programación tradicional vs. Machine Learning

Un cambio fundamental de paradigma

Programación tradicional

Entrada: Datos + Reglas explícitas

Proceso: Programador escribe instrucciones paso a paso

Salida: Resultados predecibles

Ejemplo: Si temperatura > 30°C entonces "Hace calor"

Machine Learning

Entrada: Datos + Resultados esperados

Proceso: Algoritmo descubre patrones automáticamente

Salida: Modelo que generaliza

Ejemplo: Aprender qué significa "calor" analizando miles de ejemplos

Por qué ML es revolucionario

  • "Es mucho más fácil recolectar datos que recolectar comprensión" — MIT Sloan
  • Maneja problemas demasiado complejos para reglas explícitas
  • Se adapta automáticamente a nuevos patrones sin reprogramación
  • Descubre relaciones que los humanos no pueden percibir

ML no reemplaza la experiencia humana — la aumenta procesando volúmenes de datos imposibles para análisis manual.

Aprendizaje supervisado

Aprender de ejemplos con respuestas correctas

Cómo funciona

Se proporcionan datos de entrenamiento etiquetados (entrada + salida correcta). El algoritmo aprende la relación entre entradas y salidas para predecir correctamente en datos nuevos.

Clasificación

Predice categorías discretas

  • ¿Es spam o no spam?
  • ¿Qué dígito es esta imagen?
  • ¿Aprobará o no el crédito?
  • ¿Qué enfermedad tiene el paciente?

Algoritmos: Regresión logística, árboles de decisión, Random Forest, k-NN, SVM

Regresión

Predice valores numéricos continuos

  • ¿Cuál será el precio de la vivienda?
  • ¿Cuántas ventas tendremos?
  • ¿Qué temperatura habrá mañana?
  • ¿Cuál es el valor de vida del cliente?

Algoritmos: Regresión lineal, regresión polinomial, XGBoost, LightGBM

2024-2025: Los algoritmos de Gradient Boosting (XGBoost, LightGBM) dominan en datos tabulares. Transformers comienzan a aplicarse exitosamente en datasets estructurados.

Aprendizaje no supervisado

Descubrir patrones ocultos en datos sin etiquetas

Cómo funciona

Se proporcionan datos sin etiquetas. El algoritmo busca patrones, agrupaciones o estructuras ocultas automáticamente, sin saber de antemano qué buscar.

Clustering (agrupamiento)

Agrupar datos similares

  • K-Means: crea K grupos basados en similitud
  • DBSCAN: encuentra grupos de densidad variable
  • Clustering jerárquico: crea jerarquías de grupos

Aplicaciones:

  • Segmentación de clientes
  • Detección de anomalías
  • Organización de documentos

Reducción de dimensionalidad

Simplificar datos complejos

  • PCA: análisis de componentes principales
  • t-SNE: visualización de datos complejos
  • Autoencoders: compresión con deep learning

Aplicaciones:

  • Visualización de datos
  • Compresión de información
  • Eliminación de ruido

2025: Deep Clustering integra autoencoders y técnicas generativas (VAE, GAN) con clustering tradicional para resultados superiores.

Aprendizaje semi-supervisado

Cuando etiquetar datos es costoso

El problema que resuelve

Etiquetar datos manualmente es caro y requiere mucho tiempo. El aprendizaje semi-supervisado usa pequeñas cantidades de datos etiquetados + grandes cantidades de datos sin etiquetar para entrenar modelos efectivos.

10-15%
del rendimiento supervisado con dramáticamente menos etiquetas
FreeMatch
algoritmo líder en 2025 con umbrales auto-adaptativos

Casos de uso principales

  • Imágenes médicas: solo algunas radiografías etiquetadas por expertos
  • Reconocimiento de voz: pocas transcripciones, mucho audio
  • Moderación de contenido: etiquetado humano limitado
  • Detección de fraude: pocos casos confirmados, muchas transacciones

Técnicas modernas (2025)

  • Pseudo-etiquetado: modelo genera etiquetas para datos no etiquetados
  • Consistency regularization: predicciones consistentes ante perturbaciones
  • MixMatch y FixMatch: combinan múltiples técnicas
  • FreeMatch: elimina umbrales fijos con adaptación dinámica

El semi-supervisado es ideal cuando tienes acceso fácil a datos pero etiquetar es prohibitivamente costoso.

Fase de datos: la base de todo proyecto de ML

Recolección, limpieza y exploración

Principio fundamental

Calidad sobre cantidad. Datos representativos que capturen escenarios de negocio reales son más valiosos que datasets masivos con ruido o sesgos.

Recolección

  • Identificar fuentes
  • Definir estrategia
  • APIs, web scraping, sensores
  • Bases de datos internas

Limpieza

  • Valores faltantes
  • Duplicados
  • Outliers
  • Inconsistencias

Exploración (EDA)

  • Distribuciones
  • Correlaciones
  • Visualizaciones
  • Patrones iniciales

Herramientas clave (2026)

  • Pandas: estándar de facto para manipulación de datos en Python
  • Polars: 7.7x más rápido que Pandas para CSVs grandes, evaluación lazy
  • DuckDB: analítica SQL-nativa, 2-6x más rápido que Spark
  • DVC (Data Version Control): seguimiento de cambios en datasets

60-70% del tiempo total del proyecto se consume en esta fase. Es la inversión más importante.

Fase de entrenamiento: construyendo el modelo

Frameworks, algoritmos y optimización

Frameworks dominantes (2026)

  • PyTorch: líder en investigación y producción
  • TensorFlow: fuerte en despliegue enterprise
  • Scikit-learn: esencial para ML clásico
  • Hugging Face: estándar para transformers
  • JAX: crecimiento rápido en ML de alto rendimiento

Mejores prácticas

  • Comenzar simple: baseline antes de complejidad
  • Transfer learning: aprovechar modelos pre-entrenados
  • Feature engineering: reduce tiempo 40-60%
  • Validación cruzada: evitar overfitting
  • Tracking de experimentos: MLflow, W&B, Neptune

Hiperparámetros a optimizar

  • Learning rate: velocidad de aprendizaje
  • Batch size: cantidad de ejemplos por iteración
  • Número de epochs: cuántas veces se ve todo el dataset
  • Arquitectura: capas, neuronas, funciones de activación
  • Regularización: dropout, weight decay para prevenir overfitting

Herramientas AutoML (Google Vertex AI, H2O.ai, DataRobot) pueden automatizar selección de modelos y optimización de hiperparámetros.

Fase de evaluación: midiendo el éxito

Métricas y validación rigurosa

Métricas de clasificación

  • Accuracy: % de predicciones correctas
  • Precision: de los positivos predichos, cuántos son correctos
  • Recall: de los positivos reales, cuántos detectamos
  • F1-Score: balance entre precision y recall
  • AUC-ROC: capacidad de discriminación

Métricas de regresión

  • MAE: error absoluto medio
  • RMSE: raíz del error cuadrático medio
  • R²: proporción de varianza explicada
  • MAPE: error porcentual absoluto medio

Estrategias de validación

Train/Validation/Test split: típicamente 70%/15%/15%

Validación cruzada k-fold: divide datos en k partes, entrena k veces

Validación temporal: para series de tiempo, respetar orden cronológico

Crítico: Las métricas deben alinearse con objetivos de negocio. Un modelo con 95% accuracy puede ser inútil si falla en los casos que más importan.
